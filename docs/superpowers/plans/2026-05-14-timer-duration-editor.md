# Timer Duration Editor Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers-extended-cc:subagent-driven-development (recommended) or superpowers-extended-cc:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add an inline duration editor that opens when the user re-clicks an active night or turbo timer chip, mirroring the preset-chip editor pattern. Editing the duration writes `0x0302`/`0x0303` (`pkg/breezy.SetTimerDuration`); the firmware itself restarts the running countdown.

**Architecture:** A single new server endpoint (`POST /ui/devices/{name}/timer-duration`), one new ops helper, two new client signals (`$durationEditor.<name>`, `$_durationEdit.<name>.<mode>.{hours,minutes}`), and a centred two-input editor under the existing `timer-remaining` line. The chip's first-click branch still activates the timer; the second click toggles the editor open/closed. Editor closure is enforced by a single `data-effect` that watches `$specialMode.<name>` so external deactivations and mode-switches close the editor automatically.

**Tech Stack:** Go 1.22 + templ + datastar v1 + Playwright. CSS lives in `cmd/breezyd/ui/style.css` (NOT inlined in layout).

**Spec:** `docs/superpowers/specs/2026-05-14-timer-duration-editor-design.md`

```go
// SetTimerDuration writes the configured duration for one of the
// special-mode timers: param 0x0302 (night) or 0x0303 (turbo). When
// the matching timer is currently active, the firmware restarts the
// running countdown to the new total on its own (verified against
// firmware 0.11 — see docs/superpowers/specs/2026-05-14-timer-duration-editor-design.md).
// Mode is case-insensitive. Range: hours 0..23, minutes 0..59, total ≥ 1 min.
func SetTimerDuration(ctx context.Context, c DeviceClient, mode string, hours, minutes int) error {
	if hours < 0 || hours > 23 || minutes < 0 || minutes > 59 {
		return fmt.Errorf("%w: hours 0..23, minutes 0..59, got %dh %dm", ErrInvalidArg, hours, minutes)
	}
	if hours == 0 && minutes == 0 {
		return fmt.Errorf("%w: duration must be at least 1 minute", ErrInvalidArg)
	}
	var id ParamID
	switch strings.ToLower(mode) {
	case "night":
		id = 0x0302
	case "turbo":
		id = 0x0303
	default:
		return fmt.Errorf("%w: mode must be night or turbo, got %q", ErrInvalidArg, mode)
	}
	return c.WriteParams(ctx, []ParamWrite{{ID: id, Value: []byte{byte(minutes), byte(hours)}}})
}
```

```go
// postUITimerDuration writes the configured duration for one of the
// special-mode timers. When the matching timer is currently active,
// the firmware restarts the running countdown to the new total on
// its own — no follow-up 0x0007 write needed (verified against
// firmware 0.11; see the design doc).
//
// JSON: {"mode": "night"|"turbo", "hours": 0..23, "minutes": 0..59}
//
// Hours+minutes must sum to at least 1 minute. All validation errors
// land in #global-error-banner via the SSE error envelope.
func (h *Handler) postUITimerDuration(w http.ResponseWriter, r *http.Request) {
	type req struct {
		Mode    string `json:"mode"`
		Hours   int    `json:"hours"`
		Minutes int    `json:"minutes"`
	}
	shape := func(q *req) bool {
		m := strings.ToLower(q.Mode)
		if m != "night" && m != "turbo" {
			h.uiValidationError(w, r, "", "mode must be 'night' or 'turbo'")
			return false
		}
		if q.Hours < 0 || q.Hours > 23 {
			h.uiValidationError(w, r, "", "hours must be 0..23")
			return false
		}
		if q.Minutes < 0 || q.Minutes > 59 {
			h.uiValidationError(w, r, "", "minutes must be 0..59")
			return false
		}
		if q.Hours == 0 && q.Minutes == 0 {
			h.uiValidationError(w, r, "", "duration must be at least 1 minute")
			return false
		}
		return true
	}
	postUIWriteJSON(h, w, r, shape, func(ctx context.Context, rc *recordingClient, q *req) error {
		return breezy.SetTimerDuration(ctx, rc, q.Mode, q.Hours, q.Minutes)
	})
}
```

```go
// durationEditSeed returns the inner per-mode {hours, minutes} seed for
// the duration editor's input signals. The caller wraps it in a
// {device-name: ...} map (mirrors presetSeed). Pre-poll the durations
// are 0 → both fields seed to 0 and the editor never opens until the
// user activates a timer (by which point the next poll has landed).
func durationEditSeed(v ui.DeviceView) map[string]map[string]int {
	split := func(secs int) map[string]int {
		return map[string]int{
			"hours":   secs / 3600,
			"minutes": (secs % 3600) / 60,
		}
	}
	return map[string]map[string]int{
		"night": split(v.NightDurationSeconds),
		"turbo": split(v.TurboDurationSeconds),
	}
}
```

```go
// timerClickExpr is the data-on:click body for night/turbo chips.
// Two branches keyed on wasActive (computed BEFORE any signal flip):
//
//   wasActive (re-click on the running timer): toggle $durationEditor
//     between mode and 'off'. No signal flip, no POST — the timer
//     keeps running; only the editor's open/closed state changes.
//     Mirrors the preset-chip pattern (presetClickExpr) where the
//     re-click on the active chip toggles its editor.
//
//   !wasActive (first click, or switching from the other timer): reset
//     $durationEditor.<name> = 'off' (so the OTHER mode's editor closes
//     synchronously when switching), seed $specialModeRemainingSeconds
//     from the per-mode duration signal so the countdown appears
//     instantly, then delegate to clickAction which optimistically
//     flips $specialMode + cascade-writes $power=true on activation.
//
// In-chip deactivate (the old behaviour) is gone — to stop a timer the
// user clicks any speed chip (preset/manual) or powers the device off.
// Both paths route through cascades that clear $specialMode, which the
// `.ctrl-group-timer` data-effect (see below) translates into editor
// closure. See the 2026-05-14-timer-duration-editor-design.md spec.
func timerClickExpr(name, value string) string {
	wasActive := fmt.Sprintf("$specialMode.%s === '%s'", name, value)
	activeBranch := fmt.Sprintf(
		"$durationEditor.%s = ($durationEditor.%s === '%s') ? 'off' : '%s';",
		name, name, value, value)
	seedRemaining := fmt.Sprintf(
		"$specialModeRemainingSeconds.%s = $%sDurationSeconds.%s;",
		name, value, name)
	inactiveBranch := fmt.Sprintf(
		"$durationEditor.%s = 'off'; %s %s",
		name,
		seedRemaining,
		clickAction(name,
			"specialMode",
			fmt.Sprintf("'%s'", value),
			fmt.Sprintf("/ui/devices/%s/timer", name),
			fmt.Sprintf("{mode: '%s'}", value)),
	)
	return fmt.Sprintf(
		"const wasActive = (%s); if (wasActive) { %s } else { %s }",
		wasActive, activeBranch, inactiveBranch,
	)
}
```

```go
// timerDurationChangeExpr is the data-on:change body shared by both
// inputs of the per-mode duration editor. Reads both halves of the
// $_durationEdit.<name>.<mode> signal (data-bind keeps them current
// with the input values), clamps to [0,23]/[0,59], snaps (0,0) to
// (0,1), reflects the clamped values back into the bound signals
// (so the inputs snap visually when the user types an out-of-range
// value), then mirrors the new total seconds into
// $specialModeRemainingSeconds.<name> (so the countdown line snaps
// to the new duration instantly, matching the verified firmware
// behaviour where 0x0302/0x0303 writes restart the running countdown)
// AND into $<mode>DurationSeconds.<name> (so a future fresh
// activation uses the new value before the next poll catches up).
// Finally POSTs the new duration; errors land in #global-error-banner.
func timerDurationChangeExpr(name, mode string) string {
	post := postActionExpr(
		fmt.Sprintf("/ui/devices/%s/timer-duration", name),
		fmt.Sprintf("{mode: '%s', hours: h, minutes: m}", mode),
	)
	return fmt.Sprintf(
		`let h = parseInt($_durationEdit.%s.%s.hours, 10); `+
			`let m = parseInt($_durationEdit.%s.%s.minutes, 10); `+
			`if (isNaN(h)) h = 0; `+
			`if (isNaN(m)) m = 0; `+
			`h = Math.max(0, Math.min(23, h)); `+
			`m = Math.max(0, Math.min(59, m)); `+
			`if (h === 0 && m === 0) m = 1; `+
			`$_durationEdit.%s.%s.hours = h; `+
			`$_durationEdit.%s.%s.minutes = m; `+
			`$specialModeRemainingSeconds.%s = h * 3600 + m * 60; `+
			`$%sDurationSeconds.%s = h * 3600 + m * 60; `+
			`%s`,
		name, mode, name, mode, name, mode, name, mode, name, mode, name, post,
	)
}
```
