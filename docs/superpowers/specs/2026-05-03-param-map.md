## Protocol-level findings (independent of param semantics)

- **Auth failure response:** When the password is wrong, the device replies with function `0x07` and a 2-byte payload `01 <byte>`. When the password is correct, function is `0x06`. The Go client should treat function `0x07` as `ErrAuth` rather than passing it through.
- **Discovery is unauthenticated:** Sending any request with `DEFAULT_DEVICEID` returns the device's true 16-char ID and unit type *regardless of password*. This means discovery and password-recovery via search both work even when you don't know the protocol password.
